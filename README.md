Accompanying git repository for publication involving the development of MortaLIVE as a live-dead and morphological screen assay in brain organoids


# MortaLIVE Brain Organoid Viability and Outgrowth Analysis Code

This repository contains the analysis code accompanying the manuscript:

**A Multi-Purpose Fluorescent Assay for Quantifying Viability and Morphological Screening in Human Brain Organoids**

The repository is intended to support reproduction of the manuscript's downstream quantitative analyses from processed image-analysis and plate-reader outputs. It includes two main R Markdown workflows:

1. **Neurite and glial outgrowth quantification** from Fiji/Sholl-derived profiles and processed summary tables.
2. **Bliss independence mixture additivity analysis** for Cd + Cr(VI) cytotoxicity effects using replicate-level normalized Live--Dead ratio data.

The code is provided as supplementary source code for transparency and reproducibility. It is not a turnkey image-processing pipeline; image segmentation, thresholding, and Sholl profile generation are performed upstream in Fiji/ImageJ or Yokogawa CellPathfinder, as described in the manuscript.

## Workflow 1: Neurite and glial outgrowth quantification

### Purpose

This workflow analyzes neurite and glial outgrowth from Sholl profiles. It converts Sholl intersection data into tidy long-format tables, calculates area under the curve (AUC) for each organoid/sample, performs non-parametric statistical comparisons, and generates summary plots.

The workflow supports three analysis modules:

1. **Acute metal exposure comparison**
   - Compares outgrowth across Cd, Cr(VI), Pb, and Cd + Cr(VI) mixture exposure groups.
   - Calculates Sholl AUC values for each replicate.
   - Performs pairwise Wilcoxon rank-sum tests within metal groups.
   - Generates AUC comparison plots.

2. **MortaLIVE validation analysis**
   - Compares control and MortaLIVE-exposed groups.
   - Includes both beta-tubulin III neurite outgrowth and GFAP-positive glial outgrowth.
   - Calculates raw AUC and control-normalized AUC.

3. **MortaLIVE Live-channel versus immunofluorescence comparison**
   - Compares outgrowth measurements derived from MortaLIVE Live-channel imaging with marker-based immunofluorescence measurements.
   - Supports two-group comparison of ML versus IF and three-group comparison of ML, beta-tubulin III IF, and GFAP IF.

### Expected input data

The outgrowth workflow expects one of the following input types:

1. Fiji-generated Sholl profile CSV files named like:

```text
*_Sholl-Profiles.csv
```

These files should contain at least:

```text
Radius
Inters.
```

2. Processed Excel summary tables where:

- The first column is `Radius`.
- Each additional column is a sample or replicate.
- Values are Sholl intersection counts.

Example structure:

| Radius | Ctrl_1 | Ctrl_2 | 0.0005ML_1 | 0.001ML_1 |
|---:|---:|---:|---:|---:|
| 0 | ... | ... | ... | ... |
| 10 | ... | ... | ... | ... |
| 20 | ... | ... | ... | ... |

### Important naming assumptions

The scripts infer treatment, dose, method, and replicate information from sample names. Before running the analysis, confirm that column names follow the expected patterns.

For acute metal exposure:

```text
Ctrl_1
0.32Cd_1
3.2Cr_2
10Pb_3
3.2Mix_1
```

For MortaLIVE validation:

```text
Ctrl_1
0.0005ML_1
0.001ML_1
```

For ML versus IF comparisons:

```text
sample_IF_1
sample_ML_1
```

For three-group ML/IF/GFAP comparisons, the workflow expects separate files such as:

```text
IF.xlsx
IF_GFAP.xlsx
ML.xlsx
```

If your files use different naming conventions, update the regular expressions in `01_outgrowth_quantification.Rmd` before running.

### Main processing steps

The outgrowth workflow performs the following operations:

1. Reads Fiji/Sholl CSV files or processed Excel Sholl summary tables.
2. Converts data from wide format to long format.
3. Keeps Sholl radii at 10-pixel intervals unless otherwise specified.
4. Extracts treatment, dose, method, and replicate identifiers from sample names.
5. Calculates AUC for each replicate/sample using the `MESS::auc()` function.
6. Summarizes mean and standard error of Sholl intersection curves.
7. Performs Wilcoxon rank-sum tests for group comparisons.
8. Exports processed tables and figures.

### Expected outputs

Typical output files include:

```text
data_processed/acute_metals_long_table.csv
data_processed/acute_metals_curve_summary.csv
data_processed/acute_metals_auc_per_curve.csv
data_processed/acute_metals_auc_summary.csv

results/ml_validation_gfap_auc.pdf
results/ml_vs_if_auc.pdf
```

Exact output names may differ depending on the final consolidated R Markdown file.

## Workflow 2: Bliss independence additivity analysis

### Purpose

This workflow evaluates whether the observed Cd + Cr(VI) mixture effect is consistent with Bliss independence additivity. It uses replicate-level cytotoxicity effect values derived from normalized Live--Dead ratio measurements.

The Bliss expected effect is calculated as:

```text
E_Bliss = E_Cd + E_Cr - (E_Cd × E_Cr)
```

The Bliss excess is calculated as:

```text
Bliss excess = Observed mixture effect - Bliss expected effect
```

Interaction classification is based on the 95% bootstrap confidence interval of the Bliss excess:

- CI entirely above 0: **greater than additive / synergistic**
- CI entirely below 0: **less than additive / antagonistic**
- CI overlaps 0: **additive**

### Expected input data

The workflow expects an Excel workbook with a sheet containing dose-level replicate measurements for Cd, Cr(VI), and Cd + Cr(VI) mixture groups.

Default expected file:

```text
data/bliss/Quantification Table.xlsx
```

Default expected sheet:

```text
RME quantification
```

Expected wide-format structure:

| dose | cd_1 | cd_2 | cr_1 | cr_2 | mix_1 | mix_2 |
|---:|---:|---:|---:|---:|---:|---:|
| 0.32 | ... | ... | ... | ... | ... | ... |
| 3.2 | ... | ... | ... | ... | ... | ... |
| 10 | ... | ... | ... | ... | ... | ... |
| 50 | ... | ... | ... | ... | ... | ... |

Values should be normalized Live--Dead ratio percentages. The script converts these to fractional cytotoxicity effects using:

```text
effect = 1 - (percent_LD / 100)
```

### Main processing steps

The Bliss workflow performs the following operations:

1. Reads the Excel input table.
2. Cleans column names using `janitor::clean_names()`.
3. Converts the table from wide format to long format.
4. Converts normalized Live--Dead percentages into fractional cytotoxicity effects.
5. Calculates mean Cd, Cr(VI), and mixture effects at each dose.
6. Calculates Bliss expected effect at each dose.
7. Calculates observed mixture effect and Bliss excess.
8. Uses non-parametric bootstrap resampling to estimate 95% confidence intervals.
9. Classifies each dose as additive, greater than additive, or less than additive.
10. Exports replicate-level data and Bliss summary tables.

### Bootstrap settings

Default bootstrap settings:

```r
set.seed(0831)
n_boot <- 5000
```

The random seed is included to make the bootstrap results reproducible.

### Expected outputs

Typical output files include:

```text
data_processed/bliss_replicate_level_data.csv
data_processed/bliss_independence_results.csv
results/Bliss_Independence_Mixture_Effects.xlsx
```

The exported Excel workbook should contain at least two sheets:

1. `Raw_Data`
2. `Bliss_Independence_Results`

## How to run the analyses

From the repository root, open R or RStudio and run:

```r
source("scripts/00_packages.R")
```

Then render the R Markdown workflows:

```r
rmarkdown::render("scripts/01_outgrowth_quantification.Rmd")
rmarkdown::render("scripts/02_bliss_independence_additivity.Rmd")
```

If file paths are not found, update the input path variables near the top of each R Markdown file.

Recommended path handling:

```r
repo_root <- normalizePath(".", winslash = "/", mustWork = TRUE)
input_dir <- file.path(repo_root, "data", "outgrowth")
output_dir <- file.path(repo_root, "data_processed")
results_dir <- file.path(repo_root, "results")
```

Avoid hard-coded absolute paths when preparing the repository for public release.

## Reproducibility notes

- Run scripts from the repository root unless otherwise stated.
- Do not keep Excel input files open while running the R workflows; this can interfere with file reading or writing on some systems.
- Confirm that treatment and replicate names match the expected naming patterns before running the scripts.
- The outgrowth workflow depends on upstream Sholl analysis and therefore assumes that segmentation and thresholding were completed consistently before importing data into R.
- The Bliss workflow assumes that Cd, Cr(VI), and Cd + Cr(VI) mixture effects are aligned by concentration/dose.
- For public release, remove local machine-specific paths and replace them with relative paths.
